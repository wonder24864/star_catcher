# 多轮作业检查流程

核心检查流程：第一轮检查 → 改正重检 → 渐进式求助 → 结束检查。Sprint 2。

---

## US-016: 第一轮检查

**As a** 学生
**I want to** 上传作业后看到哪些题做错了和得分
**So that** 我知道需要改正哪些题

**验收标准：**
- [ ] 上传并确认识别结果后，创建 HomeworkSession
- [ ] 显示第一轮结果：每题对/错标记 + 总得分
- [ ] **不显示正确答案、不给提示、不给解析**
- [ ] 错题用红色标记，对题用绿色标记
- [ ] 显示："15题中对12题，得分 80 分"
- [ ] 操作按钮："我改好了，再检查一次" 和 "结束检查"

**状态流转：**
```
CREATED → RECOGNIZING → RECOGNIZED → CHECKING ⟷ (多轮循环) → COMPLETED
                ↓
          RECOGNITION_FAILED
```

---

## US-017: 改正后重新检查

**As a** 学生
**I want to** 改正错题后重新提交检查
**So that** 系统可以验证我的改正是否正确

**验收标准：**
- [ ] 点击"我改好了"后，可以重新拍照或手动标记改正并输入新答案
- [ ] AI重新判断改正后的答案是否正确
- [ ] 显示新一轮结果（仍然只标对/错 + 新得分）
- [ ] 可以看到得分变化趋势（第1轮 80分 → 第2轮 93分）
- [ ] 可反复迭代，直到满分或学生选择"结束检查"
- [ ] 每轮检查结果都保留（不覆盖历史轮次）

---

## US-018: 求助功能（渐进式揭示）

**As a** 学生
**I want to** 对确实不会的题目求助
**So that** 我可以得到适当的提示帮助我理解

**验收标准：**
- [ ] 每道错题旁边有"求助"按钮
- [ ] Level 1：思路方向（考查什么知识点、解题方向）
- [ ] Level 2：关键步骤（步骤框架，无最终答案）
- [ ] Level 3：完整解析 + 正确答案
- [ ] 每个Level之间需要学生至少提交一次新答案才能解锁下一级
- [ ] 求助次数和Level记录在HomeworkSession中（家长可见）
- [ ] 家长可设置每个学生的最大求助Level：
  - 小学：默认最高Level 2（可调）
  - 初中/高中：默认最高Level 3

**解锁规则详见** `docs/BUSINESS-RULES.md` Section 7 和 `docs/adr/004-progressive-help-reveal.md`。
关键边界：空字符串不算有效答案，不能解锁下一级。

---

## US-019: 结束检查

**As a** 学生
**I want to** 结束本次作业检查
**So that** 这次检查的结果被保存

**验收标准：**
- [ ] 点击"结束检查"后，HomeworkSession 状态变为 COMPLETED
- [ ] 保存最终得分和所有轮次的历史记录
- [ ] 仍然有错题未改正的给出提示确认
- [ ] 错题自动录入错题库（去重：`contentHash` 匹配已有记录时更新 `totalAttempts`）
- [ ] 跳转到检查结果摘要页
