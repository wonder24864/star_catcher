# Knowledge Graph 用户故事

## US-031: Knowledge Graph 数据导入

**As a** 管理员
**I want to** 上传教材 PDF，让 AI 从目录结构中提取知识点树，审核后入库
**So that** 系统拥有可供 Agent 使用的知识图谱数据

**设计思路**: 中国 K-12 教材目录本身就是知识点骨架。只提取目录页/章节标题（2-5 页），不做全文分析。每本书只需 3-5K tokens，审核量小。

**验收标准：**
- [ ] 管理员可上传教材 PDF（MinIO 存储），填写教材名称/学科/年级/学段
- [ ] 系统自动识别目录页，AI 从目录结构提取知识点层级树
- [ ] 提取结果包含：知识点名称、层级关系(parent-child)、前置依赖(章节顺序)、难度估计
- [ ] 提取结果以"待审核"状态写入 KnowledgePoint 表（metadata.importStatus = 'pending_review'）
- [ ] 管理员可批量审核通过/驳回
- [ ] 支持管理员手动创建知识点（不依赖 AI 提取）
- [ ] 重复知识点检测（name + subject + grade 组合已存在时提示）

**边界条件：**
- PDF 单文件上限 50MB
- 目录提取失败时（无法识别目录页）：标记任务 FAILED，提示管理员手动录入
- AI 提取失败时标记任务状态为 FAILED 并通知管理员

**性能要求：**
- PDF 上传 + 入队：< 5s
- 目录提取 + AI 解析：< 15s（目录文本很短，单次 AI 调用）
- 知识点批量审核：< 1s / 100 条

---

### Agent 专属字段

**Skill 依赖**:
| Skill | 用途 |
|-------|------|
| extract-knowledge-points | 从教材目录文本中提取结构化知识点树 |

**Agent 循环终止条件**:
- 最大步数: 1（单次调用，目录文本作为完整输入）
- 决策停止条件: AI 返回结构化知识点列表即完成
- Token 预算: 5000 tokens

**Skill 失败恢复**:
| 失败场景 | 降级策略 |
|----------|---------|
| extract-knowledge-points 失败 | 标记任务 FAILED，管理员手动录入知识点 |
| 全部 Skill 不可用 | 标记任务 FAILED，通知管理员 AI 服务不可用 |

**成本约束**:
- 单次导入任务 Token 上限: 8000
- 每日导入任务上限: 50 次（目录提取成本低）

## US-032: Knowledge Graph 管理界面

**As a** 管理员
**I want to** 查看和编辑知识点的树形结构、关系和属性
**So that** 可以维护高质量的知识图谱数据

**验收标准：**
- [ ] 树形视图展示知识点层级（学科 → 章节 → 知识点），支持展开/折叠
- [ ] 可拖拽调整知识点层级关系（修改 parentId）
- [ ] 可编辑知识点属性（名称、描述、难度、重要性、考频）
- [ ] 可添加/删除知识点之间的关系（PREREQUISITE / PARALLEL / CONTAINS）
- [ ] 按学科、年级、学段筛选知识点
- [ ] 搜索知识点（按名称模糊搜索）
- [ ] 显示知识点的关联错题数量（QuestionKnowledgeMapping 统计）

**边界条件：**
- 删除知识点时：如有关联错题映射，提示管理员确认（软删除）
- 拖拽层级变更时：联动更新 depth 字段
- 循环依赖检测：添加 PREREQUISITE 关系前检查是否产生环

**性能要求：**
- 树形视图首次加载：< 2s（lazy load 子节点）
- 搜索响应：< 500ms
- 属性编辑保存：< 500ms
