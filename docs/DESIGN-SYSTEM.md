# Star Catcher - UI/UX 设计系统

## 1. 设计理念

采用**4 级年级自适应**策略（ADR-012）。通过 CSS 变量（视觉层）和 GradeTierProvider React Context（结构层）实现，根据学生年级自动应用对应 tier。家长端和管理端始终使用 Studio Pro 主题。

**4 Tier 映射**：
| Tier | 年级 | data-theme | 风格 |
|------|------|-----------|------|
| 1 Wonder | PRIMARY_1~3 | wonder | 魔法乐园 — 温暖、圆润、弹性动画 |
| 2 Cosmic | PRIMARY_4~6 | cosmic | 宇宙探索者 — 深色、科技感、Three.js 星空 |
| 3 Flow | JUNIOR_1~3 | flow | 极简流 — 干净、毛玻璃、spring 动画 |
| 4 Studio | SENIOR_1~3 | (default) | 专业工作室 — 数据密度优先、快速 fade |

## 3. 主题切换逻辑

- 学生端：根据年级自动选择主题（小学→Candy, 初中→Fresh, 高中→Pro）
- 家长端/管理端：始终使用 Pro 主题
- 通过 CSS 变量 + Tailwind `data-theme` 属性实现，切换无需刷新
- 学生升年级后主题自动更新
- 家长查看学生数据时保持 Pro 主题（不跟随学生主题）

## 4. 响应式断点

| 断点 | 宽度 | 场景 |
|------|------|------|
| `xs` | < 375px | 小屏手机 |
| `sm` | ≥ 375px | 标准手机（竖屏） |
| `md` | ≥ 768px | 平板竖屏 / 大屏手机横屏 |
| `lg` | ≥ 1024px | 平板横屏 / 小屏笔记本 |
| `xl` | ≥ 1280px | 桌面显示器 |

**Mobile-first 原则**：所有页面从手机竖屏开始设计，向上适配。

## 5. 核心组件规范

### 对/错标记按钮

| 主题 | 正确 | 错误 | 尺寸 |
|------|------|------|------|
| Candy | 绿色圆形, 弹性动画 | 红色圆形, 摇晃动画 | 64x64px |
| Fresh/Pro | 绿色圆角 | 红色圆角 | 48x48px |

所有主题：选中态有明确的填充色变化 + 勾选图标。

### 得分展示

- Candy: 大字号(48px)居中, 星星动画
- Fresh: 中字号(32px), 进度环
- Pro: 标准字号(24px), 数字+进度条

### 求助按钮

- Candy: "小手举起来"图标, 蓝色圆角
- Fresh/Pro: 问号图标的次要按钮
- 所有主题：与普通按钮明确区分

### 导航布局

- 手机端：底部 Tab 导航
  - 学生: 检查 / 错题 / 设置
  - 家长: 概览 / 学生 / 家庭 / 设置
- 平板/桌面：左侧 Sidebar 导航

## 6. 拍照界面

```
全屏相机预览
底部居中：圆形拍照按钮 (80x80px)
左下：相册入口图标
右下：已拍照片缩略图计数 ("2/10")
顶部：返回按钮 + 当前学科标签
相机辅助线：四角对齐框
```

## 7. 无障碍设计

- 所有可交互元素有 `aria-label`
- 支持键盘 Tab 导航
- 语义化 HTML 对屏幕阅读器友好
